使用Git新增和更新GitHub项目

发表于 2017-05-04

Git是目前最好用的版本控制系统，GitHub是基于Git实现的代码托管平台。

使用Git新增GitHub项目

  • 在GitHub上创建repository（仓库）
  • 在本地新建项目

  • 项目所在文件夹中打开Git Bash，依次输入以下命令：

    //1、建立本地仓库
git init //把这个目录变成Git可以管理的仓库

git add README.md //文件添加到仓库

git add . //不但可以跟单一文件，还可以跟通配符，更可以跟目录。一个点就把当前目录下所有未追踪的文件全部add了 

git commit -m "first commit" //把文件提交到仓库并添加comment

//2、关联github仓库
git remote add origin https://github.com/AimeeLee77/wiki_zh_word2vec.git //关联远程仓库

//3、上传本地代码推送到github仓库
git push -u origin master //把本地库的所有内容推送到远程库上

使用Git更新GitHub项目

打开本地项目所在文件夹，依次输入以下命令：

git status //查看文件状体
git add .
git commit -m "提交文件"
git push -u origin master

解决error: failed to push some refs to

在提交代码到GitHub的时候可能会报如下错误：

error: failed to push some refs to 'https://github.com/AimeeLee77/senti_analysis.git'
hint: Updates were rejected because the remote contains work that you do
hint: not have locally. This is usually caused by another repository pushing
hint: to the same ref. You may want to first integrate the remote changes
hint: (e.g., 'git pull ...') before pushing again.
hint: See the 'Note about fast-forwards' in 'git push --help' for details.

出现该错误的原因是：GitHub中的README.md不在本地代码目录中。

解决方法是：输入如下命令将代码合并，即可在本地代码目录中看到README.md文件。

git pull --rebase origin master

再执行git push -u origin master 命令即可实现本地文件的上传。

解决Branch master set up to track remote branch master from origin.

提交代码到GitHub的时候若报如下错误：

Branch master set up to track remote branch master from origin.
Everything up-to-date

原因是没有提交comment，需先输入提交评论的命令。

git commit -m "提交文件"
