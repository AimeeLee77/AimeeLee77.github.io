Day18： 20170302

1、完成了SQL和统计基础部分的学习，了解了查看统计图的方法。确定统计图可以通过位置和离散程度两大因素确定，其中曲线位置可以根据均值、中位数、众数来确定，离散程度可以根据峰度、偏度、百分位数来确定。
2、最近，在对新媒体的研究过程中学习到了推荐算法，推荐算法大致分为三类：基于内容的推荐算法、协同过滤推荐算法和基于知识的推荐算法。
基于内容的推荐算法主要用于资讯类，原理是用户喜欢和关注的Item在内容上与之有关联性的推荐给用户，可以避免冷启动的问题，弊端是可能会重复推荐。
协同过滤推荐算法主要用于电商类，原理是把具有相似兴趣用户喜欢过的商品推荐给用户；还有一种是基于Item的协同过滤算法。这两种方法都是将用户的所有数据读入到内存中进行运算。
基于知识的推荐算法主要用于广告行业，原理是构建领域本体或者建立一定的规则，进行推荐。

Day22: 20170309

1、今天学习了python的多线程基础内容。python提供实现多线程的模块之一是threading，使用最优。具体实现代码如下：
import threading 

threads=[]
t1=threading.Thread(target=,args=)
threads.append(t1)
t2=threading.Thread(target=,args=)
threads.append(t2)

if __name__='__main__':
    for t in threads:
        t.setDaemon(True)
        t.start()
    t.join

t.setDaemon(True) 表示后台进程，默认为False即前台进程。意思是当主线程执行完后，不等待子线程，直接退出，同时子线程也一同结束。

t.join 表示等待线程终止，必须等待for循环中的2个线程都结束后，才执行主进程。
这个只是多线程入门的概念，还有许多其他的内容，待慢慢学习深入了解。

2、之前情感分类问题只研究实现了数据的采集、分词处理两个内容，今天花了一下午的时间研究文本特征提取和表示的方法。

文本使用word embedding方法来表达，即文本把每个不同的词都用一个对应的向量（往往是低维向量）来表示，则同义词可以通过计算cos距离下最相似的向量来得到；

google提供了word2vec工具包，包含几种word embedding的方法来实现文本特征的表示；

2013年的word2vec工具包包含了CBOW和Skip-gram这两个模型来得到词向量。不带加速的CBOW模型是一个两层结构，通过上下文来预测中心词；不带加速的Skip-gram模型和CBOW模型和相似，二者都是通过上下文来预测中心，区别在于CBOW模型把上下文的2m个词向量求平均值揉成一个向量然后作为输入预测中心词，而Skip-gram模型则是把上下文的2m个词向量依次作为输入预测中心词。

gensim是实现word2vec的python包，依次安装numpy、scipy、gensim即可，若前两者使用pip安装失败则直接下载对应的exe安装，顺序不能弄错。

今天搞清楚了文本特征表示的基本方法和流程，并安装好了开发环境，为明天的代码实现做准备。

Day23: 20170310

1、在昨天的理论学习基础上，今天用python实现了word2vec训练模型，完成对词的表达。具体实现代码如下：

# 忽略警告
import warnings
warnings.filterwarnings(action='ignore', category=UserWarning, module='gensim')
# 引入 word2vec
from gensim.models import Word2Vec
from gensim.models.word2vec import LineSentence
import multiprocessing
# 引入日志配置
import logging,os

if __name__ == '__main__':
    logging.basicConfig(format='%(asctime)s : %(levelname)s : %(message)s', level=logging.INFO)

    # inp为输入语料
    inp = 'jd_cmts_data.txt'
    # outp1 为输出模型
    outp1 = 'jd_cmts_data.text.model'
    # outp2为原始c版本word2vec的vector格式的模型
    outp2 = 'jd_cmts_data.text.vector'

    # 训练skip-gram模型
    sentences = LineSentence(inp)
    model = Word2Vec(sentences, size=200, window=5, min_count=5,
                 workers=multiprocessing.cpu_count()-4)

    # 模型预测
    # 计算两个词的相似度/相关程度
    y1 = model.similarity(u"不错", u"好")
    print u"【不错】和【好】的相似度为：", y1
    print "--------\n"

    # 计算某个词的相关词列表
    y2 = model.most_similar(u"书", topn=20)  # 20个最相关的
    print u"和【书】最相关的词有：\n"
    for item in y2:
        print item[0], item[1]
    print "--------\n"

    # 寻找对应关系
    print u"书-不错，质量-"
    y3 = model.most_similar([u'质量', u'不错'], [u'书'], topn=3)
    for item in y3:
        print item[0], item[1]
    print "--------\n"

    # 寻找不合群的词
    y4 = model.doesnt_match(u"书 书籍 教材 很".split())
    print u"不合群的词：", y4
    print "--------\n"


    # 保存模型
    model.save(outp1)
    # 对应的加载方式
    # model_2 = Word2Vec.load("text8.model")

    # 以一种C语言可以解析的形式存储词向量
    model.wv.save_word2vec_format(outp2, binary=False)
    # 对应的加载方式
    # model_3 = Word2Vec.load_word2vec_format("text8.model.bin", binary=True)
